# ADR-008: Make Frontend Node Typings an Explicit Build Dependency

**Status**: Accepted  
**Date**: 2026-03-24  
**Deciders**: Rift Platform Team  
**Feature Branch**: `main`

---

## Context

The frontend TypeScript toolchain builds both the browser application and the Vite configuration. `frontend/tsconfig.node.json` explicitly sets `"types": ["node"]` because `vite.config.ts` runs in a Node.js context.

Until now, the frontend package manifest did not declare `@types/node` directly. That meant local builds depended on whichever transitive or peer dependency behavior npm happened to produce in a given environment. This is fragile in Docker builds, CI, and clean workspaces because the Node type package is not guaranteed to be present.

The Docker build context also included local `*.tsbuildinfo` files generated by incremental TypeScript builds. Those files are machine-local cache artifacts and should not influence container builds.

---

## Decision

Declare `@types/node` as a direct frontend `devDependency` and keep it pinned in `frontend/package-lock.json`.

Exclude `*.tsbuildinfo` files from the frontend Docker build context so container builds always recompute TypeScript state from the checked-in sources and lockfile.

---

## Alternatives Considered

### Rely on transitive or peer dependency installation

**Rejected.** Peer and transitive dependency installation behavior is not a stable contract for Rift's build. The package manifest should declare every type library required by the TypeScript configuration.

### Remove `"types": ["node"]` from `tsconfig.node.json`

**Rejected.** `vite.config.ts` is a Node-side file. Removing Node typings would hide legitimate typing requirements instead of fixing the dependency contract.

### Clean `.tsbuildinfo` inside Docker without changing `.dockerignore`

**Rejected.** This still copies local cache files into the build context and makes Docker invalidation noisier than necessary. Excluding them at the context boundary is simpler and more deterministic.

---

## Consequences

### Positive

- Frontend builds are deterministic in Docker, CI, and clean local environments.
- The TypeScript configuration and package manifest now agree on required type libraries.
- Local incremental build caches no longer leak into container builds.

### Negative / Trade-offs

- The frontend now owns one more explicit development dependency that must be updated alongside the Node.js toolchain.

### Neutral

- Runtime frontend behavior is unchanged. This decision only affects the build pipeline and developer workflow.
